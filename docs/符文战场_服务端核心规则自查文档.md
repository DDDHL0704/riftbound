# 《符文战场》服务端核心规则自查文档

版本：v1.0  
用途：交给 Codex 对现有服务端进行规则一致性自查、缺陷定位、测试补齐建议。  
适用范围：服务端规则引擎、卡牌数据、单卡实现、关键词实现、特效系统、联机状态同步、隐藏信息过滤、日志与复盘。  
规则依据：项目数据源中的《符文战场》核心规则_260330.pdf、裁判FAQ_251023.pdf、铸魂淬炼系列_官方FAQ_260114.pdf、铸魂淬炼系列_裁判FAQ.pdf、《符文战场》破限系列_裁判FAQ_260416.pdf。若规则文件、官方 FAQ、裁判 FAQ 存在冲突，应先记录冲突点；一般按“最新核心规则/官方规则 > 官方 FAQ > 最新裁判 FAQ > 较旧 FAQ”的顺序评估，但不要擅自覆盖规则文件，必须在报告中写明依据。

---

# 0. 给 Codex 的总任务

请对当前服务端进行一次“规则一致性审计”。不要只检查是否能跑通一局游戏，而要判断服务端是否足以支撑《符文战场》核心规则的完整运行，尤其是：

1. 是否有权威服务端 match state，而不是由客户端决定结果。
2. 是否能精确表达区域、对象、控制权、所属权、隐私等级、状态、临时效果、时间戳、任务队列、结算链、优先行动权、焦点、战斗、法术对决、得分、胜负。
3. 是否能阻止非法行动，并能解释非法原因。
4. 是否能正确执行所有核心行动：打出、激活、触发、确认、支付、检查合法性、结算、清理、移动、战斗、伤害分配、征服、据守、燃尽、召出、回收、放逐、摧毁、召回、贴附、卸除、创建、替换、抵挡、眩晕、展示、弃置、无效化、增益、获得资源等。
5. 是否正确实现全部关键词，以及关键词叠加、重复获得、失去关键词、依赖性关键词、权限性关键词、额外费用关键词。
6. 是否所有单卡效果与卡面文字、勘误、FAQ 一致。
7. 是否隐藏信息绝不泄漏给无权客户端。
8. 是否有足够测试证明以上内容，而不是仅凭代码结构判断。

审计完成后，请输出：

- 总体结论：READY / NOT READY / RISKY。
- P0 阻断项：会导致核心规则错误、隐藏信息泄漏、胜负错误、死锁、状态污染的缺陷。
- P1 严重项：会导致常见卡牌、关键词、FAQ 互动错误的缺陷。
- P2 改进项：日志、测试、可维护性、性能、容错方面的问题。
- 每个问题必须给出：规则依据、代码位置、现象、最小复现场景、建议修复方向、建议新增测试。
- 对未能确认的项目，标记为 UNKNOWN，不要默认为通过。

# 1. 自查评分标准

## 1.1 结论等级

READY：核心规则路径、常见关键词、结算链、法术对决、战斗、得分、隐藏信息、单卡效果测试均有明确证据支撑，未发现 P0 阻断项。

RISKY：主流程能跑，但存在未覆盖的规则窗口、疑似不符合官方文本的关键词/单卡实现、清理/FEPR/战斗/隐私等模块测试不足。

NOT READY：存在任一 P0 缺陷，例如隐藏信息泄漏、非法动作可执行、结算链死锁、战斗伤害错误、得分胜负错误、起手/卡组/区域根本不符合规则。

## 1.2 严重程度定义

P0：必须先修。包括但不限于：隐藏信息泄漏；服务端信任客户端结果；无法表达四种回合状态；结算链 FEPR 错误；清理循环错误；战斗或法术对决会卡死；非法行动能通过；得分/胜负错误；卡组/开局不符合规则；重复触发或漏触发导致局势明显错误。

P1：上线前应修。包括但不限于：关键词边界错误；FAQ 交互错误；单卡文本部分实现偏差；资源支付或额外费用边界错误；伤害来源、摧毁来源、目标失效、替换效果、效果层顺序错误。

P2：可排期优化。包括但不限于：日志不够清晰；错误提示不够精确；缺少复盘视角；测试夹具不够好；性能或序列化不够稳定；规则引用缺失。

# 2. 必须先做的代码结构审计

## 2.1 服务端权威性

检查项：

- 客户端只能提交行动意图，不能提交行动结果。
- 洗牌、随机战场、先后手、抽牌、随机顺序、随机目标等均由服务端决定。
- 所有 action 均有服务端合法性检查。
- 服务端拒绝非法 action 时，不改变 match state。
- 所有状态变更均由同一规则引擎入口产生，不允许绕过规则引擎直接改对象字段。
- 每个行动有递增序号或事务 ID，可以处理重发、乱序、并发双击。
- 同一时间只允许处理一个游戏行动，符合规则 303.1、303.2。

P0 风险信号：

- 客户端传入“已抽到的牌”“已造成的伤害”“已得分”“战斗结果”。
- 有 API 直接写入对象状态，例如直接 setScore、setController、setDamage 而不走规则流程。
- WebSocket handler 与规则引擎各自修改 match state。
- 同一个 match 同时处理两个 action，没有锁、队列或事务。

## 2.2 状态快照与隐藏信息过滤

检查项：

- 服务端保存全知状态；向每名玩家发送的是过滤后的视角状态。
- 对手手牌只显示数量，不显示 cardId、instanceId 对应的卡面、效果、排序。
- 对手主牌堆和符文牌堆只显示数量，不显示顺序、卡名、实例 ID 到卡面的映射。
- 对手正面朝下待命牌只显示卡背、所属待命区、控制者、数量/占位，不显示卡名和效果。
- 隐秘、私密、公开三类信息严格对应规则 128；公开区域状态均可见，私密/隐秘区域只向有权者显示。
- 复盘结束后若提供全知视角，必须只在游戏结束后开放。

P0 风险信号：

- 为了前端渲染方便，把对手手牌完整 cardId 发给客户端但让前端隐藏。
- 把牌堆顺序作为 snapshot 发给双方。
- 面朝下待命牌在对手视角仍可由 ID 推断具体卡。
- 日志中泄漏了抽到的具体牌、换走的具体牌、牌堆顶内容。

## 2.3 数据版本与规则版本

检查项：

- match 记录使用的规则版本、卡牌数据版本、勘误版本、FAQ 补丁版本。
- 所有卡牌 effect definition 能追溯到卡牌文本版本。
- 对同名卡的勘误、功能性勘误、规则阐明有版本标记。
- 禁止混用旧文本和新规则而不标记。

P1 风险信号：

- 卡牌数据只有图片和文本，没有结构化 effect。
- 结构化 effect 与文本不一致且无测试。
- FAQ 中被勘误的卡仍用旧效果。

# 3. 卡组构筑与开局流程自查

规则重点：103、111–119、480。

## 3.1 卡组合法性

必须检查：

- 每名玩家有 1 张传奇卡，游戏开始进入传奇区域。
- 主牌堆至少 40 张，包含 1 张选定英雄单位，以及其他单位、装备、法术。
- 选定英雄必须是英雄单位，且英雄标签与传奇匹配。
- 主卡组同名最多 3 张；选定英雄计入同名数量。
- 专属卡总数限制：卡组中一共只能包含三张与传奇卡具有相同英雄标签的专属卡牌；专属卡不是英雄单位，不得放入英雄区域。
- 主牌堆卡牌必须符合传奇符文特性；多特性卡必须同时被传奇符文特性允许。
- 符文牌堆必须为 12 张符文，且符合传奇符文特性。
- 1v1 决斗中每名玩家提供 3 张战场；需要多个战场时，不得包含同名战场超过 1 张；战场是否需符合符文特性按模式和卡牌规则检查。
- 唯我关键词在卡组构筑中生效：每个卡组只能包含一张该名称的卡。

必须测试：

- 少于 40 主牌拒绝准备。
- 符文不是 12 张拒绝准备。
- 选定英雄标签不匹配拒绝准备。
- 主牌同名 4 张拒绝准备。
- 选定英雄加主牌同名超过 3 拒绝准备。
- 多特性卡只在传奇特性完全覆盖时可加入。
- 同一角色不同名称允许各 3 张。
- 专属卡总数超过 3 拒绝准备。
- 唯我同名 2 张拒绝准备。
- 3 张战场里出现同名战场拒绝准备。

## 3.2 1v1 决斗准备流程

必须检查：

- 双方传奇放入传奇区域。
- 双方选定英雄放入英雄区域。
- 每名玩家从自己的 3 张战场中随机选 1 张进入本局，剩余 2 张移除且本局不使用。
- 中央战场数量为 2。
- 主牌堆、符文牌堆分别洗牌，且顺序作为隐秘信息。
- 公平随机决定回合顺序。
- 双方各抽 4 张牌。
- 按回合顺序进行手牌调度；每名玩家最多选择 2 张手牌搁置，抽等量新牌，然后回收搁置的牌。
- 起始玩家开启第一个回合。
- 第二个行动的玩家，在自己首个召出阶段额外召出 1 张符文，也就是常规 2 张 + 额外 1 张；若符文不足则按召出规则尽可能召出。

必须测试：

- 起手调度只能选择 0、1、2 张，不能选择 3 张。
- 调度搁置牌不会进入废牌堆，不会被对手看见，最后回收到主牌堆。
- 调度抽牌不足时是否按抽牌/燃尽规则处理；若你的规则引擎有特殊开局处理，必须说明依据。
- 后手首个召出阶段额外召出只触发一次，不能每回合触发。
- 后手如果跳过或替换首个召出阶段，额外召出的处理必须有明确规则依据。

# 4. 区域、对象、所属权、控制权、状态

规则重点：106–110、120–189。

## 4.1 区域模型

服务端必须能表达以下区域：

- 基地：每名玩家一个，是位置；玩家控制的常驻牌和符文通常位于基地；公开。
- 战场区域：包含多处战场；每处战场是单独位置；战场和其上的常驻牌公开。
- 待命区域：每处战场一个子区域；默认容量 1；不是位置；正面朝下待命牌为私密；只有控制该战场时才能放入或占据。
- 传奇区域：每名玩家一个；不是位置；传奇为游戏物体，不能移除、移动、移位。
- 结算链：非场地区域；公开；同一时间只能有一条。
- 废牌堆：每名玩家一个；公开；无序。
- 英雄区域：每名玩家一个；公开；选定英雄可从此处按正常打出规则打出；常规方式不能回到此区，除非该区域为空且规则指示。
- 主牌堆区域：每名玩家一个；隐秘；顺序隐秘。
- 符文牌堆区域：每名玩家一个；隐秘；顺序隐秘。
- 放逐区域：每名玩家一个；公开；无序。
- 手牌：每名玩家一个；私密；数量公开。

必须测试：

- 每张卡/指示物任意时刻只处于一个区域，或是作为已贴附对象共享顶部卡牌位置；不得出现幽灵副本。
- 卡牌进入非场地区域时，不得进入其他玩家对应非场地区域，应改入所属者对应区域。
- 从场地区域到非场地区域，或非场地区域到其他区域，临时状态停止追踪：伤害清除、增益移除、临时关键词失效等。
- 废牌堆、放逐区无序，不应用顺序影响效果，除非效果明确指定。

## 4.2 游戏物体与卡牌类别

必须检查：

- 单位、装备、法术、符文、战场、传奇、指示物、结算链技能、增益/状态标记等都能作为游戏物体被规则引用。
- “卡牌”在卡面效果中通常指主牌堆卡牌；符文、传奇、战场在执行游戏物体技能和效果时不被视为卡牌，除非规则上下文特指。
- 指示物不是卡牌；指示物进入非场地区域时按规则消失。
- 战场不是常驻牌，不会被打出/召出/摧毁/移动。
- 传奇不是常驻牌，不会被摧毁/移动，可以拥有技能。
- 符文不属于主牌堆卡牌，不是常驻牌；回收时回到符文牌堆。

P0/P1 风险信号：

- 用一个 Card 类承载所有对象且没有区分“卡面”“游戏物体实例”“结算链技能实例”。
- 指示物被放进手牌或废牌堆后仍存在。
- 战场被当成可摧毁常驻牌。
- 传奇被移动或移除。

## 4.3 所属者与控制者

必须检查：

- 所属者是将卡带入游戏的玩家，游戏中一般不变。
- 打出卡牌者成为该游戏物体控制者；法术、技能、常驻牌、符文控制权可被引用。
- 技能控制者默认是技能来源控制者。
- 战场控制权是二元状态：由某玩家控制或不受控制。
- 战场技能由战场控制者控制；未受控制战场的技能触发时，由回合玩家负责添加至结算链和做决定；若规则需要控制者，视回合玩家为控制者。
- 战场技能文本中的“你”通常指战场控制者；若无控制者，“你”不指代任何玩家，相关指示应无视，除非勘误/FAQ 另有说明。

必须测试：

- 控制权改变不改变所属者。
- 对手控制我方卡牌后，非场地区域回归仍进入我的对应区域。
- 未受控制战场触发技能，由回合玩家排序和决策。
- 战场处于争夺状态时，控制权不会改变，原控制者仍控制战场技能和待命区，直到争夺/战斗结束后的相应处理。

## 4.4 状态与临时效果

必须检查：

- 活跃/休眠是场上非法术游戏物体状态；已休眠不能再次休眠作为费用，除非规则明确允许。
- 单位默认休眠进场；装备默认活跃进场；急速等可改变单位进场状态。
- 伤害单独记录在单位上；伤害不是游戏物体；回合结束和战斗清理移除。
- 眩晕是二元状态；下个结束步骤开始时失去；眩晕单位不为战斗伤害贡献战力，但仍需承受致命伤害才被摧毁。
- 增益是游戏物体；每名单位同时只能有一个增益；提供 +1 战力；可消耗；单位离开游戏时增益移除。
- 经验是玩家公开数值，不是游戏物体，无上限，不与队友共享。
- 强力单位按当前战力是否 5 或以上判断；场外单位按自有战力判断。

# 5. 回合状态机、阶段、任务队列

规则重点：300–326。

## 5.1 四种回合状态

服务端必须显式表达：

- 普通开环：没有法术对决/战斗，且无结算链。
- 普通闭环：没有法术对决/战斗，但有结算链。
- 法术对决开环：有法术对决/战斗，且无结算链。
- 法术对决闭环：有法术对决/战斗，且有结算链。

必须检查：

- 若有法术对决或战斗，回合处于法术对决状态。
- 若有结算链，回合处于闭环状态。
- 普通开环默认只有回合玩家拥有优先行动权且可打出普通时机卡牌/技能。
- 法术对决状态默认只有迅捷或反应可被打出/激活。
- 闭环状态默认只有反应可被打出/激活。
- 迅捷与反应只是权限，不改变卡牌/技能自身其他限制。

P0 风险信号：

- 用一个 boolean isStackOpen 表达所有状态，无法区分法术对决与普通状态。
- 在法术对决里允许打普通法术。
- 在闭环里允许打只有迅捷没有反应的法术。
- 没有焦点概念，只用 activePlayer。

## 5.2 优先行动权与焦点

必须检查：

- 任意时间最多一名玩家拥有优先行动权。
- 只有拥有优先行动权的玩家可以进行自决行动。
- 任意时间最多一名玩家拥有焦点。
- 法术对决开环中，拥有焦点且拥有优先行动权的玩家可行动。
- 焦点玩家让过优先行动权时保留焦点。
- 普通状态没有焦点。
- 获得焦点同时获得优先行动权。
- 闭环中，结算链下一个项目控制者、让过传递等能正确授予优先行动权。

必须测试：

- 当前无优先行动权时，任何自决行动都被拒绝，但限定行动/被要求选择可执行。
- 结算链最新项目控制者在合适时点获得优先行动权。
- 法术对决中打出法术并结算后，焦点按规则传递；但由触发式技能或获得资源技能开启的结算链不应错误传递焦点。
- 获得资源技能确认/结算不传递优先行动权与焦点。

## 5.3 回合阶段

必须检查：

回合开始阶段：

- 唤醒阶段：回合玩家将其控制的所有能变活跃的游戏物体变为活跃。
- 开始阶段开始时触发相应效果。
- 得分计算步骤：回合玩家据守其控制的所有战场。
- 召出阶段：回合玩家从符文牌堆召出 2 张符文；符文不足则有多少召多少。
- 抽牌阶段：回合玩家抽 1 张；若主牌堆无牌则燃尽后仍要抽；抽牌阶段结束时清空所有玩家符文池。

主阶段：

- 结构不限，回合玩家可进行任意数量合法自决行动。
- 自决行动后可能插入结构化阶段：战斗、法术对决、结算链、清理。
- 玩家无行动或选择结束时进入回合结束阶段。

回合结束阶段：

- 结束步骤处理“回合结束阶段”相关效果。
- 失效步骤启动回合结束特殊清理，插入为所有单位移除伤害、所有“本回合内”效果同时失效。
- 若失效期间有项目经历 FEPR，则返回失效步骤起点重新开始。
- 回合队列下一个玩家成为回合玩家。

必须测试：

- 回合开始变活跃不会激活不能变活跃的对象。
- 据守只在开始阶段得分计算步骤发生。
- 每个战场每回合最多得 1 分。
- 抽牌阶段后所有玩家符文池清空，不只是当前玩家。
- 回合结束时所有玩家符文池清空。
- “本回合内”效果同时失效，不应按任意顺序逐个失效。

## 5.4 任务队列与清理触发点

必须检查：

清理会在以下情况变为未决任务：

- 开环/闭环状态转换后。
- 游戏阶段转换后。
- 待处理项目添加至结算链后。
- 待处理项目变为合法结算链项目后。
- 结算链项目从结算链移除后。
- 任意数量游戏物体进入或离开场地后。
- 任意游戏物体状态改变后。
- 移动完成后。

清理执行原则：

- 清理过程中不结算结算链项目。
- 清理中可以添加新的待处理项目，但合法项目不执行，优先行动权与焦点不授予/传递。
- 结算链项目结算期间不发生清理；满足清理条件则列为未决任务。
- 如果清理期间又满足清理条件，前一次清理完成后立即再清理，直到清理不再改变游戏状态。

P0 风险信号：

- 只在回合结束清理，不在移动、状态改变、对象进离场后清理。
- 清理过程中结算触发效果。
- 清理中触发的新清理被吞掉。
- 状态改变导致单位致命伤害后未立即清理摧毁。

# 6. 结算链与 FEPR 流程

规则重点：327–340、349–359、398–406。

## 6.1 结算链基本结构

必须检查：

- 打出卡牌或激活/触发技能时，创建或使用同一条结算链。
- 同一时间只能有一条结算链。
- 被打出的卡牌与技能在成为已确认结算链项目前，先作为待处理项目加入结算链。
- 只要结算链有项目，回合处于闭环。
- 待处理项目必须完成确认流程，才能成为已确认项目。
- 常驻牌确认后离开结算链进场并执行规则文本；法术确认后留在结算链，等待让过与结算。
- 技能确认后通常像法术一样成为结算链项目；获得资源技能确认后立即结算。

## 6.2 FEPR：确认、执行、让过、结算

必须检查：

确认：

- 若有待处理项目，控制者必须按加入结算链的先后顺序完成打出/激活步骤，直到成为已确认项目或离开结算链。
- 确认不会让过优先行动权。
- 若结算链为空，游戏以开环继续；若发生在法术对决期间，回合顺序下一名玩家获得焦点。
- 若结算链不为空且无待处理项目，结算链最新效果控制者获得优先行动权。

执行：

- 拥有优先行动权的玩家可以打出符合时机的卡牌、激活符合时机的技能、或让过。
- 闭环默认只有反应能加入。
- 新加入项目后返回确认。

让过：

- 所有玩家均在没有新增项目的情况下让过，才进入结算。
- 否则优先行动权按回合顺序传递。

结算：

- 结算链最新项目先结算。
- 若结算链为空，游戏开环；若在法术对决中，焦点按回合顺序传给下一名玩家。
- 若结算链仍有待处理，返回确认。
- 若结算链不空且无待处理，最新效果控制者获得优先行动权。

必须测试：

- 法术 A 上链，对手反应 B，上方 B 先结算，然后 A 再等待让过/结算。
- 结算期间产生触发，触发作为待处理加入，但不在当前结算中插队。
- 结算链中多个待处理按加入顺序确认。
- 新反应加入后，之前的“双方已让过”记录失效，需要重新让过。
- 获得资源技能立即确认/结算，不改变焦点/优先行动权。
- 触发式技能费用无法支付时，按规则从结算链移除，不视为无效化。

## 6.3 打出卡牌流程

必须检查 6 步：

1. 将要打出的卡牌从所在区域移除并放到结算链；回合进入闭环；项目待处理。若当前有其他效果正在结算或有未决任务，先继续完成那些流程。
2. 做相关选择：模式、位置、目标、移动终点、分摊目标等。单位默认可打到控制者基地或控制者控制的战场；装备默认只能打到玩家基地；法术/技能目标必须有效。能够选择“任意数量”或“最多”时允许选择 0 个目标。
3. 确定总费用：基础费用、替代费用、无视费用、强制额外费用、可选额外费用、增费、减费；费用不能低于 0；法盾是强制额外费用；急速/回响等是可选额外费用。
4. 支付费用：支付符能、法力、非标准费用；可用反应获得资源技能在此时产生资源。
5. 检查合法性：目标仍合法；不会创造非法状态；当前时机合法。若非法，回滚打出过程中采取的所有行动。
6. 完成确认：常驻牌进场/法术成为已确认结算链项目/技能成为结算链项目。

P0/P1 风险信号：

- 先支付费用再选目标，导致法盾费用计算错。
- 目标合法性只在宣告时检查，不在确认时检查。
- 非法打出没有回滚已支付资源、移动区域、消耗费用。
- 法术目标全部失效时被“无效化”，而不是仍结算并无视相应目标/指示。
- 常驻牌错误地一直留在结算链上等待让过。

## 6.4 法术结算与目标失效

必须检查：

- 法术从上到下执行规则文本。
- 结算过程中其他法术/技能不能确认或结算；产生的触发/待处理项目等当前法术全部指示完成后再处理。
- 即使部分或全部目标不合法，法术依旧结算；非法目标不受影响。
- 无法遵循的指示被无视；能执行的尽量执行。
- 如果某张卡所有指示无法实现，它仍可成功打出并完成结算，但不产生效果。
- “无法”优先于“可以”。
- 需要查询目标信息时，若目标已不合法，应按规则处理，不得使用过期对象错误结算。
- 区域变更前信息回溯、关联指示、替换后续指示等必须有通用实现或卡牌级测试。

必须测试：

- 法术选目标后目标离场，法术仍结算但该目标不受影响。
- 多目标法术一个目标失效，其他合法目标仍受影响。
- 所有目标失效，不退还费用，不触发“被无效化”。
- 卡牌所有效果都无法执行，仍视为打出成功。
- 禁止效果和允许效果冲突时，禁止优先。

# 7. 技能系统

规则重点：360–406、742。

## 7.1 被动技能

必须检查：

- 被动技能影响常规游戏流程，是条件、限制、声明或持续效果。
- 常驻牌被动技能通常只在场上启用。
- 场外卡牌被动技能必须自行说明适用条件，例如改变自身费用。
- 被动技能可以产生替换效果、禁止效果、费用修正、权限修正、效果层修正。
- 卡牌贴附后，其规则文本未激活，但效果文本会添加到顶部卡牌规则文本。

## 7.2 主动技能

必须检查：

- 主动技能格式为“费用：效果”。
- 主动技能使用结算链，流程类似打出卡牌。
- 默认只能在控制玩家回合和开环状态激活；单位/装备的主动技能通常在控制者回合主阶段普通开环，且不在法术对决/战斗期间。
- 带迅捷/反应的主动技能获得额外时机权限，但仍遵守自身其他限制。
- 支付费用前必须确保费用能完成；休眠作为费用时，已休眠对象不能支付。
- 获得资源类技能确认后立即结算。

必须测试：

- 已休眠单位不能支付 [E] 主动技能费用。
- 对手回合普通开环不能激活无反应/迅捷技能。
- 法术对决中可激活迅捷技能。
- 闭环中可激活反应技能。
- 获得资源反应技能可在支付费用步骤使用，且不传递焦点/优先行动权。

## 7.3 触发式技能

必须检查：

- 条件满足后，触发式技能像主动技能一样作为待处理项目加入结算链。
- 触发条件在可能引发它的事件处理完毕后判定。
- 常驻牌触发通常只有在场上时判定；离场同时触发的特殊情况按规则处理，例如绝念。
- “你可以选择”触发由控制者决定是否放入结算链。
- 触发式技能的指示文本中的费用在确认时支付；必须支付才可确认加入结算链；可拒绝支付并移除，不视为无效化。
- 同时多个触发：同一控制者自己排序；不同控制者从回合玩家开始按回合顺序排序。
- 初始战斗结算链的进攻/防守触发有特殊排序：焦点/进攻方先，非防守方按回合顺序，防守方最后；该规则优先于一般触发排序。
- 进攻触发/防守触发每场战斗只在单位首次获得对应身份时触发一次。
- 指目标效果在法术/技能完成确认后触发。
- 征服/据守效果在相应得分行动发生时触发；即便原本得分被撤销或替换，FAQ/规则对应触发仍需检查。

必须测试：

- 多个同控触发让玩家排序。
- 多个不同控触发按回合玩家顺序。
- 战斗初始结算链中，防守方触发总在最后加入。
- 进攻单位失去再获得进攻方身份，同一战斗不重复触发进攻触发。
- 触发式技能确认时目标已无合法选项，则从结算链移除，不视为无效化。
- 触发式技能有费用且玩家拒绝支付，移除且不视为无效化。

## 7.4 替换效果、延迟技能、关联技能、内嵌式触发

必须检查：

替换效果：

- 描述通常含“将”“改为”；会改变事件或指示。
- 同一替换效果对同一事件只生效一次。
- 同一事件有多个替换效果，由受影响物体所属者、受影响玩家，或未受控制战场的回合玩家决定顺序。
- 多个同时事件逐个处理；同一控制者可排序；多控制者按回合顺序。
- “每回合限一次”替换效果可选择是否生效，选择不生效则本回合未生效。
- 燃尽是替换效果。
- 抵挡是延迟替换效果。

延迟技能：

- 由法术/技能创建，不依赖创建来源是否还在场，除非规则另有限制。
- 延迟关联技能引用受影响对象时，只要对象在合法区域内窗口持续。

关联技能：

- 组件必须来自同一游戏物体卡面或同一来源赋予。
- 引用受同组技能影响的对象时，只能与该关联组对象互动。

内嵌式触发：

- 会创建一个或多个待处理结算链项目。
- “进行 N 次”应放入 N 次。
- 多个待处理项目按顺序创造，遵守待处理/确认流程。

# 8. 资源、符文、费用

规则重点：131、160–167、356–357、429–430。

## 8.1 资源模型

必须检查：

- 法力用于支付数值费用，无特性。
- 符能用于支付特性费用，有特性。
- [A] 作为费用可用任意特性符能支付。
- [C] 默认代表卡牌主要特性；若卡牌无特性或多特性，则按 [A] 处理。
- 符文池只是概念储存库，不是实体对象；但服务端需要准确记录未消耗资源。
- 获得资源后先进入符文池，才能支付费用。
- 每名玩家抽牌阶段结束，以及回合结束时，所有玩家符文池清空。

## 8.2 基础符文

必须检查：

- 基础符文具备两个技能：[E]：[反应] — 获得 [1]；回收此牌：[反应] — 获得 [C]。
- 回收基础符文获得的符能与被回收符文特性相同。
- 符文被回收回到符文牌堆，而非主牌堆。
- 符文不是常驻牌。

## 8.3 费用计算

必须检查：

- 基础费用来自印刷费用，即使打出时费用被改变，其他“判定费用”的效果通常看印刷费用。
- 替代费用、无视费用先改变基础费用。
- 强制额外费用必须加入，例如法盾。
- 可选额外费用只有玩家选择才加入，例如急速、回响。
- 增费、减费顺序与规则一致。
- 针对部分费用的减费先于总费用减费。
- 费用不能降到 0 以下。
- 减费可以减少额外费用；可选额外费用即使被减到 0 也视为已支付。
- 非标准费用按任意顺序支付；被替换效果替代的费用仍可视为已支付。
- 费用支付失败要回滚确认流程。

必须测试：

- 法盾目标被选多次，每次目标选择施加强制额外费用。
- 可选额外费用被减到 0，仍视为已支付。
- [A] 可用任意符能支付。
- [C] 在多特性卡上按 [A]。
- 打出费用被修改后，查询卡牌费用仍返回印刷费用，除非文本明确查总费用/支付费用。

# 9. 基础游戏行动

规则重点：413–439。

## 9.1 抽牌与燃尽

必须检查：

抽牌：

- 默认从主牌堆抽到手牌。
- 抽 X 张牌时，若主牌堆不足：有多少抽多少，执行燃尽，再抽剩余数量。

燃尽：

- 当玩家试图将一张或多张卡牌从主牌堆移动到任意其他区域，且移动数量超过主牌堆剩余数量，执行燃尽。
- 查看或展示主牌堆卡牌数量不足时，只尽可能查看/展示；如果后续行动无法执行则无视后续，不因此燃尽，除非确实尝试移动超过剩余数量。
- 燃尽步骤：尽可能完成原指示；将废牌堆回收到主牌堆；选择一名对手，使其获得 1 分；完成触发燃尽的剩余行动。
- 主牌堆和废牌堆都空时，可能连续燃尽，每次令一名对手得 1 分，直到胜利或能完成动作。
- 第一个燃尽按顺序处理后获得的分数不能被替换或阻止。

必须测试：

- 主牌堆 0、废牌堆有牌，抽 1：燃尽、对手得 1、回收、抽 1。
- 主牌堆 1，抽 3：抽 1、燃尽、回收、继续抽 2。
- 主牌堆空，查看牌堆顶 1 张：不燃尽。
- 主牌堆空，展示牌堆顶并要把展示牌移动：若没有展示到牌，则后续无视，不燃尽。
- 主牌堆空且废牌堆空，抽 1：可能循环燃尽至胜利或规则终止；不得死循环无胜负检查。

## 9.2 回收、弃置、展示

必须检查：

- 主牌堆卡牌回收回主牌堆底部；符文回收回符文牌堆底部。
- 每名玩家回收自己的卡牌到自己的牌堆，与谁执行回收无关。
- 同时回收 2 张以上主牌堆卡牌，以随机顺序放到底部。
- 同时回收 2 张以上符文，由拥有者安排顺序放到底部。
- 洞察超过主牌堆数量时尽可能洞察，不燃尽。
- 弃置是从手牌进入废牌堆，不打出、不执行卡面文本；作为费用时必须能完成。
- 展示是临时公开信息，不改变区域；玩家自愿分享私密信息不视为“展示”。

## 9.3 造成伤害、抵挡、额外伤害

必须检查：

- 法术、单位、技能、效果可以对单位造成伤害。
- 战斗中分配伤害不是造成伤害，但分配完成后同时造成，来源视为单位。
- 伤害被抵挡时，替换为造成减少后的伤害；不低于 0；全部抵挡则视为未造成伤害。
- 抵挡作用于下一次符合条件的伤害。
- 额外伤害对造成伤害行动生效；多个额外伤害相加；若实际未造成伤害，额外伤害不生效。
- 多目标伤害中，额外伤害分别加到每个目标；分摊伤害时额外伤害加到可分摊总量。

## 9.4 眩晕、无效化、增益、放逐、摧毁

必须检查：

眩晕：

- 已眩晕单位不能再次眩晕。
- 下个结束步骤开始时失去眩晕。
- 眩晕单位不贡献战斗伤害战力。

无效化：

- 被无效化的卡牌/技能不产生效果并移出结算链。
- 被无效化的卡牌不视为被打出。
- 不退还费用，包括额外费用。
- 只有效果指示时才能无效化。

增益：

- 每名单位最多一个增益，重复给予不增加第二个。
- 增益是游戏物体，可引用、计算、指定目标。
- 消耗增益只能消耗自己控制单位上的增益。

放逐：

- 从任意区域直接放到放逐区域。
- 放逐不是摧毁，不是弃置。

摧毁：

- 摧毁是常驻牌从场上进入废牌堆，不属于移动。
- 只有被摧毁前在场上区域，才视为被摧毁。
- 主动摧毁来自效果/费用；被动摧毁来自致命伤害或状态结果。
- 摧毁来源可以有多个；技能是摧毁来源时，创造该技能的游戏物体也视为来源。
- 绝念在常驻牌因摧毁指示或清理移入废牌堆前作为待处理加入结算链，并记录相关信息。

## 9.5 贴附、卸除、创建、替换

必须检查：

贴附：

- 贴附会让一张或多张卡变为已贴附状态，另一张成为顶部卡牌。
- 已贴附卡牌规则文本未激活，效果文本添加到顶部卡牌规则文本，战力加成应用到顶部卡牌。
- 已贴附卡牌仍是场上卡牌，保有类型/标签，可被目标选择。
- 顶部卡牌移动时，已贴附卡牌随之移动。
- 顶部卡牌从场地区域到非场地区域时，已贴附卡牌卸除并留在当前区域；若卸除装备出现在战场，下次清理召回。
- 顶部卡牌控制权变化不改变已贴附卡牌控制权。

卸除：

- 只对已贴附卡牌有效。
- 卸除后规则文本恢复，效果文本未激活，顶部卡牌失去相应技能和战力加成。

创建：

- 创建直接在指定区域生成不存在的游戏物体。
- 未指定区域时，按类型进入适当区域：法术到结算链、符文到基地、传奇到传奇区、战场到战场区域。
- 创建的战场默认未受控制。

替换：

- 替换不是放逐。
- 被替换的卡牌或指示物放到放逐区域，并视为已被替换而非放逐。
- 指示物进入放逐区域停止存在。
- 换回时，指示物停止存在，原始卡牌回到指示物占据位置并继承当前效果和状态。

# 10. 移动、召回、战场控制

规则重点：144、187、440–452。

## 10.1 标准移动

必须检查：

- 标准移动只能在玩家主阶段，不能在闭环状态，不能在法术对决或战斗期间。
- 标准移动费用是让单位变为休眠状态；已休眠单位不能支付。
- 玩家可同时执行多个单位标准移动；终点必须与宣告一致；费用同时支付。
- 单位可从所属基地移动到战场。
- 单位可从战场移动回所属基地。
- 游走允许单位通过标准移动从一处战场到另一处战场。
- 单位不得移动到已存在两名其他玩家控制单位的战场；1v1 中通常不会触发该多人限制，但引擎若支持多人必须正确。
- 移动立即完成，不使用结算链，无法被反应。
- 移动完成后进行清理。

P0 风险信号：

- 移动进入结算链且允许反应。
- 已休眠单位仍可标准移动。
- 移动后未触发清理，导致战斗/法术对决不发生。

## 10.2 召回

必须检查：

- 召回是将常驻牌不使用移动地转移至其所属基地。
- 召回不是移动，不触发移动触发。
- 限制/阻止移动的效果不影响召回。
- 召回不影响被召回常驻牌的状态，除非来源另有说明；伤害、休眠、增益、已应用效果层变动保持。
- 未贴附装备出现在战场时，下次清理召回控制者基地。

## 10.3 战场控制与争夺

必须检查：

- 当非当前战场控制者的玩家所控制单位以移动或其他方式出现在该战场，且战场尚未受到争夺，该战场进入争夺状态。
- 争夺状态只是规则内部临时状态，卡牌效果不能引用。
- 在确立/重新确立控制权前，战场保持争夺。
- 战场进入争夺后，法术对决或战斗结束时，仍有单位的玩家建立控制权。
- 战场上若有玩家单位且该单位未处于战斗状态，只要该玩家仍在该战场留有单位，便保持控制。
- 正在战斗或法术对决时，除非步骤另有指示，战场控制权不改变。
- 如果玩家在某战场没有单位且回合处于开环，且没有战斗/法术对决，则该战场变为未受控制。
- 玩家失去战场控制后，下次清理移除该战场待命区中相关待命牌；但战斗/法术对决中争夺未结束前，不提前移除。

必须测试：

- 单位进入无人控制空战场，进入争夺并开启非战斗法术对决。
- 单位进入对手控制且有对手单位的战场，进入争夺并待发生战斗。
- 法术对决中原控制者单位离开，原控制者仍保持战场控制直到争夺结束。
- 法术对决中待命牌不会因暂时没有控制者单位而提前移除。
- 争夺结束且控制权改变后，下次清理移除非新控制者的待命牌。

# 11. 法术对决

规则重点：316.5、341–348。

必须检查：

- 法术对决是开环机会窗口，供玩家交替打出符合时机的法术/技能互动。
- 发生战斗时即发生法术对决，且战斗法术对决是战斗的下属阶段。
- 无控制者战场进入争夺状态时，标记非战斗法术对决待发生；非战斗法术对决进行期间，其他玩家控制的单位出现在该战场会使战斗待发生。
- 法术对决开始时，令战场进入争夺状态的玩家获得焦点。
- 法术对决期间，拥有焦点玩家可打出符合时机卡牌、激活符合时机技能或让过。
- 如果所有玩家依次让过一次，法术对决结束并进行清理。
- 否则焦点按回合顺序传给下一名玩家。
- 法术对决中最后一个结算链项目结算且回到开环时，焦点按回合顺序传递；但由触发式技能或获得资源技能开启的结算链不按此方式传焦点。

必须测试：

- 进攻方开启法术对决获得焦点。
- 双方让过一次后法术对决结束。
- 玩家打出迅捷法术后结算完，焦点正确传递。
- 玩家打出反应法术形成闭环，只有反应可继续响应。
- 触发式技能创建结算链后，结算完焦点不错误传递。
- 非战斗法术对决中出现敌对单位，转入/标记战斗待发生逻辑正确。

# 12. 战斗

规则重点：455–461。

## 12.1 战斗启动

必须检查：

- 清理时，若结算链上无项目，同时某战场有待发生战斗，且其他战场没有正在进行的法术对决或战斗，则发生战斗。
- 同一战场存在两名敌对玩家控制的单位但战斗步骤未启动，则战斗待发生。
- 多个战场同时待发生战斗，由回合玩家选择先结算哪场。
- 若待发生战斗在战斗步骤前不再待发生，则不结算。
- 战斗只在两名且只有两名玩家控制的单位之间发生。

## 12.2 战斗法术对决步骤

必须检查：

- 战斗开启时，确定进攻方与防守方。
- 令战场进入争夺状态的玩家为进攻方，并获得进攻方身份。
- 未令战场进入争夺状态的玩家为防守方，并获得防守方身份。
- 战场处于争夺状态时，进攻方单位获得进攻方身份；防守方单位获得防守方身份；之后新出现在该战场的相关单位也在行动/效果后获得对应身份。
- 进攻方获得焦点。
- 若确立攻防导致触发式技能待处理，则添加战斗初始结算链。
- 初始战斗结算链关闭后，焦点不会转移。

必须测试：

- 进攻/防守触发进入初始结算链顺序正确。
- 初始结算链结算完焦点仍按 FAQ/规则保持，不错误传给下一玩家。
- 进攻/防守身份随控制者、位置、战斗状态修正。

## 12.3 战斗伤害步骤

必须检查：

- 如果进攻方和防守方都有单位留存在战场上，才结算战斗伤害。
- 计算所有进攻方单位战力之和。
- 计算所有防守方单位战力之和。
- 从进攻方开始，每名玩家将等同于其战力总和的伤害分配给对方单位。
- 分配伤害不是造成伤害；全部分配完后同时造成。
- 眩晕单位不贡献战力。
- 战力低于 0 时，战斗伤害分配时视为 0，但实际值仍保留供其他效果计算。
- 分配时，在其他单位承担伤害前，必须先给一个单位分配致命伤害。
- 尚有其他可分配单位时，不得给某单位分配超过致命伤害所需最小值。
- 必须遵守所有伤害分配要求和限制。
- 多个相同优先级要求，分配玩家可任意排序。
- 同一单位上互斥分配要求，分配玩家选择应用一个。
- 无法被造成伤害的单位，任何伤害不视为致命，不受强制伤害分配约束，可在分配致命伤害时忽略。
- 抵挡会影响致命伤害计算；全部抵挡时无论多少伤害都不视为致命。

必须测试：

- 8 战力攻击两个 1 战力单位，不能把 8 点全部给一个 1 战力单位。
- 壁垒单位与非壁垒单位同控时，必须先给壁垒单位分配致命伤害。
- 两个壁垒单位同优先级，可任选顺序。
- 壁垒与另一个要求互相排斥时，分配玩家可选择应用其中一个。
- 无法被造成伤害单位可被忽略，不强迫先分配。
- 眩晕单位不贡献战力但仍可被分配/承受伤害。
- 战斗伤害来源被视为单位，不是“分配”本身。

## 12.4 战斗结算与战斗清理

必须检查：

- 战斗伤害后跳过 FEPR，进入结算步骤。
- 战斗结算步骤进行战斗特殊清理。
- 战斗清理插入：为所有单位移除伤害；若该战场仍存在防守方单位，则召回位于该战场的所有进攻方单位。
- 战斗清理中的清除伤害发生在常规清理特定步骤之间，绝念等触发时点不能错。
- 判定战斗结果：仅一方有单位存活则该方赢得战斗；双方都有或双方都无则无结果；无结果且双方仍有单位则标记待发生战斗。
- 若该位置无待发生法术对决或战斗，则此处有单位存活的玩家确立控制；清除争夺；无单位则未受控制；移除非控制者待命牌；建立控制权且本回合该战场未得分则征服。
- 战斗结束：移除所有进攻方/防守方身份；所有“本次战斗”效果同时失效。

必须测试：

- 1 战力进攻 5 战力防守，伤害结算后进攻方阵亡，防守方有 1 点战斗伤害但在战斗清理中移除；绝念触发造成后续伤害时，防守方伤害状态符合 FAQ。
- 防守方仍在战场，进攻方单位被召回。
- 攻防双方都存活，战斗无结果并标记新待发生战斗。
- 双方都无单位，战斗无结果且不建立控制。
- 只有进攻方存活，进攻方确立控制并可能征服得分。
- 只有防守方存活，防守方保持/确立控制，不给进攻方征服分。

# 13. 得分与胜负

规则重点：462–467、480、650–652。

必须检查：

- 1v1 决斗获胜得分为 8。
- 得分是通过夺取或保持战场控制而获得分数。
- 征服：玩家获得一处本回合尚未得分战场的控制权。
- 据守：玩家在回合开始阶段保有一处战场控制权。
- 每个回合，每名玩家只能通过征服和据守从每个战场获得 1 分。
- 玩家得分时，根据当前得分最多获得 1 分；致胜分有额外限制。
- 若玩家当前总分为比获胜得分低 1 分或更高，尝试通过据守得分，则直接获得致胜分。
- 若玩家当前总分为比获胜得分低 1 分或更高，尝试通过征服得分，则必须在本回合通过任一方法在每个战场都已获得得分，才能获得致胜分。
- 得分后触发对应战场上的征服/据守技能；每名玩家每回合每战场仅触发一次。
- 清理发生时，如果玩家分数大于或等于获胜得分且高于所有对手，则获胜。
- 认输：玩家可随时认输；1v1 中剩余玩家获胜；认输玩家移除流程应无泄漏/死锁。

必须测试：

- 玩家 7 分据守一个战场，直接到 8 并在清理胜利。
- 玩家 7 分征服一个战场但本回合未在所有战场得分，不应获得致胜分。
- 玩家 7 分本回合两个战场都得过分，再征服/完成条件可得致胜分。
- 同一战场同一玩家本回合已征服得分，之后据守/再夺回不能重复得分。
- 双方同时达到 8 但不高于所有对手，不应错误判胜；直到清理中满足领先才胜。
- 燃尽给对手分数后，清理胜负检查时机正确。
- 认输时，对手获胜；结算链/焦点/优先行动权不再等待认输玩家。

# 14. 效果层、持续效果、时间戳

规则重点：468–475。

必须检查：

- 服务端不能简单按“效果应用发生顺序”永久写死最终战力/关键词；需要能重新评估效果层。
- 效果层顺序：改变特质；改变技能/规则文本；加减计算。
- 复制效果复制可复制特质，除非效果指定。
- 已贴附卡牌效果文本中的技能添加在改变技能层；已贴附卡牌战力加成在加减计算层。
- 加减计算中正值/增值先应用，负值/减值后应用。
- 限定条件在效果持续期间保持快照的情况要处理。
- 同层效果存在依赖关系时，按依赖处理；无依赖则按时间戳。
- 规则文本未激活时失去时间戳；恢复激活时确立新时间戳。
- 持续效果失效与应用是独立事件；效果移除后需重算。

P1 风险信号：

- 战力修改直接加到字段上，不保留效果来源和持续时间。
- 贴附/卸除后不重算顶部卡牌技能和战力。
- 复制效果复制了临时效果或没有复制印刷规则文本。
- 多个持续效果顺序不稳定，重放/复盘结果不同。

# 15. 关键词逐项自查

规则重点：800–826，以及 700 系附加规则。

## 15.1 通用关键词机制

必须检查：

- 关键词是可被其他效果引用的特性。
- 关键词可由效果提供或移除，需有持续时间。
- 如果提供/移除关键词未说明持续时间，则一般只要对象留在场上或当前区域保持有效；具体按文本。
- 重复获得关键词的行为必须按各关键词定义处理，不可统一去重或统一叠加。
- 提示文本不产生规则功能。
- 依赖性关键词下的技能在条件未满足前未激活，但仍视为存在，可供引用和判定。
- 权限性关键词只提供打出/激活权限，不改变卡牌效果本身。

## 15.2 急速

必须检查：

- 急速是单位技能，可选额外费用。
- 打出该单位时可额外支付 [1] 和 1 点符能，使其以活跃状态进场。
- 单特性单位急速符能必须同特性；无特性或多特性用 [A]。
- 只有打出流程中可支付；场上不能补交。
- 场上没有功能。
- 多个急速只生效一次。
- 急速直接改变进场状态，不是先休眠再激活，不触发“进入活跃状态”相关技能。

## 15.3 迅捷

必须检查：

- 迅捷允许卡牌/技能在任意玩家回合法术对决期间打出/激活。
- 迅捷不允许在闭环状态响应，除非同时有反应。
- 迅捷只提供权限，不改变其他限制。
- 条件性获得迅捷的卡/技能，只要有可能满足可先进入结算链，但检查合法性时仍未满足则回滚。

## 15.4 反应

必须检查：

- 反应允许卡牌/技能在任意玩家回合闭环状态期间打出/激活，并适用迅捷相关权限。
- 反应不改变其他限制；反应单位仍只能打到合法位置。
- 条件性获得反应同样在检查合法性时验证。
- 待命状态正面朝下或从正面朝下打出时获得反应，此属性不属于公开信息。

## 15.5 强攻与坚守

必须检查：

- 强攻 X：如果我是进攻方，[M]+X；默认 X=1。
- 坚守 X：如果我是防守方，[M]+X；默认 X=1。
- 成为进攻方/防守方身份期间持续生效。
- 多个强攻/坚守叠加。

## 15.6 绝念

必须检查：

- 绝念是触发式技能关键词，格式为阵亡后执行效果。
- 常驻牌被摧毁并送入废牌堆时触发。
- 若“被摧毁”被替换为召回等没有送入废牌堆，则不触发。
- 因摧毁指示或清理移入废牌堆前，绝念作为待处理项目加入结算链，并记录位置、属性及相关信息。
- 每个绝念分别触发；控制者排序。

## 15.7 法盾

必须检查：

- 法盾 X：对手控制且将我选为目标的法术或技能，每将我选作一次目标，其费用增加 X 点任意符能；默认 X=1。
- 法盾施加强制额外费用。
- 同一个法术/技能多次选择同一法盾对象，按“每将我选作一次目标”加费。
- 多个法盾是否叠加按各实例规则处理，不能简单只取一个。

## 15.8 游走

必须检查：

- 游走允许单位使用标准移动从一处战场移动到另一处战场。
- 不移除其他标准移动选项。
- 不提供额外移动机会或额外技能。
- 多个游走只生效一次。

## 15.9 待命

必须检查：

- 待命是执行待命自决行动的前提，可出现在法术、单位、装备上。
- 在你回合开环状态下，若此牌在手牌或英雄区域，可以选择你控制的一处战场，将它正面朝下置入该战场待命区。
- 正面朝下布置待命不属于打出，不开启结算链。
- 从待命状态打出卡牌会开启结算链。
- 待命常驻牌必须打出到其待命的战场；包括待命装备，此规则优先于装备只能打到基地的常规限制。
- 待命法术/常驻牌打出效果若导致你打出单位，该单位必须打到该待命战场。
- 除待命限制外，卡牌其他技能和指示照常。
- 拥有待命的牌也可正常时机正常费用打出。
- 多个待命只生效一次。
- 待命牌正面朝下及从正面朝下打出时获得反应，且该属性不是公开信息。
- 每个待命区默认容量 1；控制战场才能占据/放入；失去控制后下一次清理移除。

## 15.10 鼓舞

必须检查：

- 鼓舞是依赖性关键词：如果你本回合已打出过其他卡牌，则此牌获得后续文本。
- “其他卡牌”必须不同于带鼓舞的卡。
- 同一回合只需打出一张其他卡，即满足你控制所有带鼓舞卡的条件。
- 条件满足时依赖性技能生效；未满足时未激活但仍可被引用。

## 15.11 壁垒与后排

必须检查：

壁垒：

- 战斗伤害步骤中，在为其他同控制者但不带壁垒的单位分配伤害前，必须先给壁垒单位分配致命伤害。
- 仍须一次给一个单位分配致命伤害。
- 同控多个壁垒单位，可任选顺序。
- 多个壁垒只生效一次。

后排：

- 战斗伤害步骤中，必须先给其他同控制者且不带后排的单位分配致命伤害，才能给后排单位分配。
- 同控多个后排单位，在其他单位满足后可任选后排顺序。
- 多个后排只生效一次。

必须测试壁垒与后排冲突、相同优先级、无法受到伤害、抵挡全部伤害的组合。

## 15.12 瞬息、预知

瞬息：

- 常驻牌关键词；触发条件为该常驻牌控制者的开始阶段开始。
- 多个瞬息只触发一次。

预知：

- 常驻牌进场触发：查看主牌堆顶部一张牌，可以选择回收。
- 每个预知分别触发。
- 多个预知之间若未回收且没有其他过程，看到的应仍是同一张顶牌。
- 洞察/查看不足不燃尽。

## 15.13 装配、灵便、百炼

装配：

- 装配是主动技能关键词，通常在武装装备上。
- 激活装配支付费用，选定单位为目标，并将该装备贴附到目标单位。
- 装配费用可能包含资源和非资源费用，可能被文本改变。
- 触发式技能因支付费用或做选择触发时，作为待处理项目加入。
- 多个装配等于多个主动技能，可分别激活。
- 即使规则文本未激活，装备是否拥有装配仍可被引用。

灵便：

- 灵便装备自带反应，允许用反应时机打出并贴附。
- 简写为反应 + 当你打出此牌时，将其贴附于你控制的一名单位上。
- 多个灵便不分别触发，除第一个外无效果。

百炼：

- 百炼是单位打出效果。
- 选择一件你控制的武装，允许以减少后的费用支付其装配费用，将其卸除并贴附到具有百炼的单位上。
- 费用确定视同该装配技能被激活并选择百炼单位，但装配技能并未实际激活，百炼单位也未被装配技能选择。
- 若无法支付装配费用、无法卸除、无法贴附，则不能执行。
- 多个百炼分别触发，可选择不同目标；同一目标也分别结算。

## 15.14 回响

必须检查：

- 回响是法术可选额外费用。
- 打出法术时可支付回响费用，以额外执行一次该法术效果。
- 多个回响费用可分别选择支付；每个回响费用仅可支付一次。
- 因回响额外执行时，所需选择必须在打出卡牌“做出相关选择”步骤的常规时点做出；额外执行选择可与初始选择不同。
- 无论执行多少次，该法术仅视为打出一次。

## 15.15 伏击、狩猎、等级、唯我

伏击：

- 单位可被打出到有己方单位的战场。
- 这是打出流程选择位置时增加有效位置。
- 若确认完成前所选位置没有己方单位，伏击权限失效；若无其他权限，打出非法并回滚。
- 多个伏击只生效一次。

狩猎：

- 当我征服或据守一处战场时，我的控制者获得 X 经验；默认 X=1。
- 多个狩猎叠加。

等级：

- 如果控制者经验不少于 N，此牌获得后续文本。
- 控制者变化后，按新控制者经验重新判断。
- 经验低于 N 后依赖性技能立即未激活。

唯我：

- 构筑限制，不是对局效果。
- 同名每卡组只能 1 张。
- 若同时为专属，专属总数仍可三张组合，但每种名称最多一张。

# 16. 单卡与特效实现审计

## 16.1 所有单卡必须建立“文本—结构化效果—测试”三方一致性

Codex 需要遍历所有卡牌，生成一份单卡审计表：

- cardId / 中文名 / 英文名 / 系列 / 稀有度 / 类型 / 特性 / 费用 / 战力 / 标签 / 关键词。
- 当前卡面文本与勘误文本是否一致。
- 结构化 effect definition 是否覆盖卡面每一条独立技能、关键词、指示、模式、费用、目标、持续时间、触发条件、替换条件。
- 是否有单元测试或集成测试覆盖该卡至少一个成功场景和一个失败/边界场景。
- 是否在 FAQ 中出现；若出现，是否有回归测试覆盖 FAQ 问答。
- 若存在“人工处理”“未实现”“TODO”“特殊硬编码”，标记 P1 或 P0，视影响决定。

## 16.2 单卡审计重点

检查每张卡时必须逐字回答：

- 这张卡是单位、装备、法术、符文、战场、传奇、指示物，还是多类别？
- 这张卡的每个关键词是否按核心规则实现，而不是按提示文本猜测？
- 是否存在模式选择？模式是否在打出/确认时选择，且确认后不可更改？
- 是否存在目标选择？目标是否符合 355 的目标规则？
- 是否存在“任意数量”“最多”？是否允许 0 目标？
- 是否存在“选择”但不是目标的例外？是否正确区分目标与非目标选择？
- 是否存在分摊伤害？是否在确认时选择目标，结算时分配伤害？
- 是否存在额外费用、替代费用、减费、增费、法盾？顺序是否正确？
- 是否存在指示文本中的费用？法术在结算时支付，触发式技能在确认时支付，这一点是否区分？
- 是否存在“如果你如此做”“以此”“若不能”等关联指示？前置指示失败后后续是否正确处理？
- 是否存在“本回合内”“本次战斗”“直到离开场地”等持续时间？是否在正确时点失效？
- 是否创建延迟触发、延迟替换、持续效果、效果层修改？是否记录来源、控制者、时间戳？
- 是否改变控制权、移动、召回、贴附、卸除、替换、创建？是否触发正确清理？
- 是否引用“我”“此牌”“此法术”“此处”“你”“友方”“敌方”“落单”“一对一”“战斗中”等术语？解析是否符合规则？
- 是否涉及隐藏信息查看、展示、抽牌、回收？是否不泄漏给对手？
- 是否涉及燃尽？是否只在移动主牌堆卡牌数量超过剩余时燃尽，而查看/展示不足不燃尽？
- 是否涉及战场控制、征服、据守、最终分限制？是否正确？
- 是否存在“无法”和“可以”冲突？禁止是否优先？

## 16.3 自动化单卡审计建议

Codex 应检查是否已有以下测试或可生成测试：

- 每张卡至少一个“能被合法打出/激活/触发”的基础测试。
- 每张卡至少一个“非法时机/非法目标/资源不足”的拒绝测试。
- 每张卡如果有 FAQ，至少一个 FAQ 回归测试。
- 每张带关键词的卡，至少覆盖该关键词通用测试。
- 每张有勘误的卡，测试使用勘误后文本而非旧文本。
- 每张涉及隐藏信息的卡，测试对手视角不会泄漏。
- 每张涉及随机的卡，测试随机由服务端 seed 控制并可复盘。
- 每张涉及复杂效果层的卡，测试重算结果而非增量叠加结果。

# 17. FAQ 回归测试清单

以下不是全部 FAQ，而是最低必测样例。Codex 还应自动扫描项目数据源中的所有 FAQ，把每个 Q&A 映射成测试或人工审计项。

## 17.1 裁判FAQ_251023 必测

- 所有效果都无法执行：卡仍成功打出并完成结算，不产生效果。
- “任意数量”“最多”目标：允许选择 0 个目标；选择 0 时可无目标打出。
- 法术/技能结算期间要求打出另一张卡：被要求打出的卡先以待处理效果加入结算链，完成当前效果所有指示后再确认。
- 同一结算期间产生多个待处理效果：按加入顺序确认，不得提前让后生成单位成为先确认技能的合法目标。
- 同时多个触发：同控排序，多控从回合玩家开始。
- 战斗初始结算链多玩家触发：进攻方/焦点相关顺序，防守方触发最后。
- 进攻触发/防守触发同一战斗只触发一次。
- 触发式技能有费用：确认时可支付或拒绝；拒绝则移除，不视为无效化。
- 获得资源技能不会传递优先行动权和焦点。
- 初始结算链创建时，令战场进入争夺的玩家是焦点玩家；初始结算链结算完焦点不转移。
- 初始结算链后活跃玩家/优先行动权由最新效果控制者决定。
- 战场处于争夺/战斗法术对决期间，原控制者即便无单位仍维持控制。
- 未受控制战场技能触发，由回合玩家负责添加和决策。
- 战场技能中的“你”指控制者；无控制者则相应指示无对象并无视，除非勘误。
- 清理期间不结算结算链；新待处理可加入；清理重复到状态不再变化。
- 战斗清理插入移除伤害等特殊步骤，且清除争夺状态不属于清理本身。
- 战斗法术对决中失去控制者单位，不提前移除待命牌；争夺清除后的下一次清理才检查移除。
- 战斗伤害分配不是造成伤害；分配完成后同时造成；来源视为单位。
- 不能在尚有其他单位时给一个单位分配超过致命所需伤害。
- 多个相同优先级伤害分配要求，玩家任意排序。
- 互斥伤害分配要求，分配玩家选择一个应用。
- 无法被造成伤害的单位可在致命分配中忽略。
- 伤害来源和摧毁来源都可能有多个来源。
- “无法”高于“可以”。
- 主牌堆为空时查看/展示不会燃尽；尝试移动超过剩余数量才燃尽。

## 17.2 铸魂淬炼官方 FAQ/裁判 FAQ 必测方向

Codex 应重点扫描并生成测试：

- 官方 FAQ 中所有勘误卡牌是否使用最新文本。
- 官方 FAQ 中规则阐明是否已纳入 effect definition。
- “赢得战斗时”的发生时点是否与战斗结果判定一致。
- 贴附、装配、百炼、灵便、武装相关问题是否按最新规则处理。
- 回响支付、回响多次执行、回响选择目标与法术仅视为打出一次。
- 经验、等级、强力、狩猎相关互动。
- 战场控制、征服/据守、最后一分限制相关互动。
- 费用来源、符能特性、法盾、额外费用和减费相关互动。
- 替换、换回、映像/复制类效果、持续效果层相关互动。

## 17.3 破限系列裁判 FAQ 必测方向

Codex 应重点扫描并生成测试：

- 官方规则更新后产生变动的卡牌问题是否已改到最新实现。
- 空主牌堆、从牌库打出、燃尽、回收相关卡牌是否正确。
- 特殊打出位置、伏击/战场打出权限、控制者变化、打出到战场是否正确。
- 复制/映像/创建指示物/替换类卡牌是否按最新设计意图。
- 多个相同传奇/基兰/Z 型驱动等回合时序与延迟效果是否正确。
- 惩戒等在我方回合/对方回合时机差异是否正确。

# 18. 联机同步、日志、复盘

## 18.1 WebSocket / 实时同步

必须检查：

- 每个 match 有唯一权威状态。
- 每个 action 有序号，服务端按序处理。
- 客户端重连后收到基于其身份过滤后的完整快照。
- 断线玩家轮到行动时有明确超时策略；超时动作由服务端生成并写日志。
- 观战者只能看到公开信息，或按房间设置延迟/全知；默认不得看到隐藏信息。

## 18.2 日志

必须检查：

- 公开日志与私密日志分离。
- 抽牌公开日志只显示“抽 1 张”，私密日志向抽牌玩家显示具体卡。

- 目标、费用支付、让过、结算、清理、得分、燃尽、战斗伤害、摧毁来源必须可追溯。
- 每条日志关联 actionId、stateVersion、规则步骤或卡牌来源，方便复盘与调试。

## 18.3 复盘

必须检查：

- 复盘可通过 action log 重放得到同一最终状态。
- 随机事件有 seed 或记录，不依赖客户端。
- 公开复盘不泄漏隐藏信息；全知复盘只在游戏结束后开放。
- 复盘中能看到每次规则窗口：谁有优先行动权、谁有焦点、结算链内容、待处理项目、清理任务。

# 19. 测试体系要求

## 19.1 单元测试

必须覆盖：

- 卡组校验。
- 区域移动与隐藏信息过滤。
- 费用计算与支付。
- 目标选择与失效。
- 结算链 FEPR。
- 触发排序。
- 清理循环。
- 移动/召回。
- 战场控制。
- 法术对决。
- 战斗伤害分配。
- 得分/胜负。
- 每个关键词。
- 每张复杂卡/FAQ 卡。

## 19.2 集成测试

必须至少覆盖完整小局面：

1. 标准开局到第一个主阶段。
2. 后手首个召出阶段额外召出符文。
3. 单位进入空战场，非战斗法术对决，双方让过，建立控制，征服得分。
4. 单位进入对手有单位战场，战斗法术对决，双方让过，分配伤害，清理，判定控制。
5. 法术上链，对手反应，反应先结算，原法术再结算。
6. 法术目标离场后仍结算并只影响合法目标。
7. 触发式技能在结算期间产生，当前法术完成后才确认。
8. 燃尽导致对手得分并可能获胜。
9. 最后一分限制：据守可直接致胜，征服需本回合所有战场得分。
10. 认输立即移除玩家并判负。

## 19.3 属性测试/模糊测试

建议 Codex 检查是否有如下不变量测试：

- 所有卡牌实例任意时刻只在一个区域或一个贴附结构中。
- 公开视角永不包含对手隐藏 cardId。
- 分数不因同一战场同一玩家同一回合重复征服/据守增加。
- 任意时刻最多一名玩家有优先行动权。
- 任意时刻最多一名玩家有焦点。
- 同一时间最多一条结算链。
- 清理循环最终稳定或触发胜负，不得无限循环。
- 回合状态必为四种之一。
- 无 action 处理期间，match state 可序列化/反序列化并保持一致。
- 重放 action log 得到相同 final state。

## 19.4 快照测试

每个规则关键场景应保存：

- 行动前全知状态。
- 行动意图。
- 行动后全知状态。
- 玩家 A 视角状态。
- 玩家 B 视角状态。
- 公开日志。
- 私密日志。
- 可执行 actions 列表。

Codex 应查找这些测试是否存在；若不存在，列为测试缺口。

# 21. 最终验收门槛

服务端要被认为“足够支撑核心规则运行”，至少必须满足：

- 无 P0 缺陷。
- 1v1 决斗完整开局、回合、法术对决、战斗、得分、胜负可由服务端自动推进。
- 所有隐藏信息过滤测试通过。
- FEPR、清理、战斗、得分、燃尽都有集成测试。
- 全部关键词有通用测试。
- FAQ 涉及卡牌有回归测试或明确人工审计结论。
- 所有卡牌 effect definition 与当前文本/勘误建立映射。
- action log 可重放。
- 非法 action 不改变状态，并返回明确原因。

如果以上任一项无法证明，结论不得写 READY，只能写 RISKY 或 NOT READY。
