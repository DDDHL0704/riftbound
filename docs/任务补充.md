总目标：
把项目推进到可以稳定完成 1v1 双人联机对战的产品级状态。

硬性原则：
1. 前端不得自行裁决游戏规则。
2. 前端只展示服务端 authoritative snapshot 中允许展示的信息。
3. 前端只提交服务端 ActionPrompt / LegalAction 支持的动作。
4. 所有合法性、隐藏信息、随机、洗牌、结算链、法术对决、战斗、伤害分配、清理、得分、胜负都必须由服务端权威处理。
5. 如果前端发现接口缺失，不得伪造规则结果；必须补齐服务端接口。
6. 服务端缺失或审查文档指出的 P0/P1 规则问题，必须按官方规则补齐实现和测试。
7. 最终复审达到无 P0/P1 阻断问题后，才允许标记 complete。

1. 首页 / 大厅
2. 卡牌图鉴

8. 战场控制、争夺状态、待命区、单位状态展示
9. 移动、战斗、伤害分配、得分、燃尽、胜负结算展示
10. 日志、规则提示、错误提示、断线重连、复盘入口

验收必须通过：
1. 后端 full test 全绿。
2. 前端 build 无错误。
3. TypeScript / lint 无 P0/P1 问题。
4. Browser smoke 通过。
5. Playwright 或等效 E2E 完成双浏览器 1v1 对战主流程。
6. 隐藏信息测试通过：对手手牌、牌堆顺序、面朝下待命牌不得出现在非授权客户端。
7. 服务端与前端状态不得分叉。
8. 完成 completion audit 报告后，才允许把 goal 标记 complete。

Browser smoke 最低流程：
1. 玩家 A 创建房间。
2. 玩家 B 加入房间。
3. 双方选择合法卡组。
4. 双方准备。
5. 房主开始游戏。
6. 双方完成起手调整。
7. 游戏进入第一回合。
8. 当前玩家召出符文、抽牌、打出单位。
9. 单位移动到战场。
10. 触发战场争夺或战斗。
11. 进入法术对决 / 结算链窗口。
12. 双方可以让过。
13. 结算正常推进。
14. 可以结束回合。
15. 对手回合正常开始。
16. 断线重连后状态恢复。
17. 至少完成一处战场得分。
18. 投降或达到胜利条件后正常结算。
